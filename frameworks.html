Testing a Website


During one of my courses, I created a test suite to test a local page of login. In it, I assert values of of the webpage bieng via CSS selector like checkboxes, blank spaces, login, the privacy policy, simulate drag-and-drop of files, make an HTTP request and even use a custom command. All the tests were written in JavaScript.




    Website:
  1. Checks The Application Title
  2. Fills In The Required Fields And Submits The Form
  3. Displays Error Message When Submitting The Form With An Email With Invalid Formatting
  4. The Phone Field Remains Empty When Filled With A Non-Numeric Value
  5. Displays Error Message When The Phone Number Becomes Mandatory But Is Not Filled In Before Sending The Form
  6. Fills In And Clears The First Name, Last Name, Email And Phone Fields
  7. Displays Error Message When Submitting The Form Without Filling In The Required Fields
  8. Successfully Submits The Form Using A Custom Command
  9. Selects A Product (Youtube) By Text
  10. Selects A Product (Mentorship) By Its Value
  11. Selects A Product (Blog) By Its Index
  12. Selects The Type Of Service "Feedback".
  13. Ticks Each Type Of Service
  14. Check Both Checkboxes, Then Uncheck The Last One
  15. Selects A File From The Fixtures Folder
  16. Select A File Simulating A Drag-And-Drop
  17. Select A File Using A Fixture That Has Been Given An Alias
  18. Verify That The Privacy Policy Opens In Another Tab Without A Click
  19. Accesses The Privacy Policy Page By Removing The Target And Then Clicking On The Link
  20. Tests The Privacy Policy Page Independently
  21. Displays And Hides Success And Error Messages Using .Invoke
  22. Fills The Text Area Using The Invoke Command
  23. Makes An Http Request
  24. By Finding A Cat

Testing a Website


During an exercise, I took the liberty to test and create a test suite for a streaming data website. While the test suite might seem "small", the objective was to show negative and positive test cases - while some had to pass, others needed to fail in order to show what was the correct behaviour. While the test itself is recorded with Selenium IDE, the test cases are written in Python.





    Website:
  1. Login Quix
  2. Languages Filter Broken
  3. Case Sensitive Broken
  4. API Deployment Stuck
  5. Unable to Edit Code
  6. Alphabetical Selector Broken
  7. Adding Slack Notification

...plus manual tests!

          class Test3CaseSensitive():
            def setup_method(self, method):
              self.driver = webdriver.Chrome()
              self.vars = {}
            
            def teardown_method(self, method):
              self.driver.quit()
            
            def test_3CaseSensitive(self):
              # Test name: 3-CaseSensitive
              # Step # | name | target | value
              # 1 | open | webiste
              self.driver.get("https://www.quix.io/")
              # 2 | setWindowSize | 2044x1392 | 
              self.driver.set_window_size(2044, 1392)
              # 3 | click | css=.mat-list-item:nth-child(2) > .mat-list-item-content | 
              self.driver.find_element(By.CSS_SELECTOR, ".mat-list-item:nth-child(2) > .mat-list-item-content").click()
              # 4 | click | css=div > .text-truncate:nth-child(1) | 
              self.driver.find_element(By.CSS_SELECTOR, "div > .text-truncate:nth-child(1)").click()
              # 5 | click | css=.mat-list-item:nth-child(2) > .mat-list-item-content | 
              self.driver.find_element(By.CSS_SELECTOR, ".mat-list-item:nth-child(2) > .mat-list-item-content").click()
              # 6 | click | id=mat-input-0 | 
              self.driver.find_element(By.ID, "mat-input-0").click()
              # 7 | type | id=mat-input-0 | transformation
              self.driver.find_element(By.ID, "mat-input-0").send_keys("transformation")
              # 8 | click | css=.w-100:nth-child(2) | 
              self.driver.find_element(By.CSS_SELECTOR, ".w-100:nth-child(2)").click()
              # 9 | assertText | id=mat-input-0 | RESULTS
              assert self.driver.find_element(By.ID, "mat-input-0").text == "RESULTS"
